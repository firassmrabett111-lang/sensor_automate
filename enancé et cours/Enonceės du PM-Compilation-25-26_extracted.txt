Année : 2025-2026
Durée : 6 semaines Équipes : 2 étudiants
Contexte du Projet
La métropole "Neo-Sousse 2030" souhaite déployer une plateforme intelligente de gestion des
données urbaines. Cette plateforme doit non seulement collecter et analyser des données, mais
aussi permettre aux gestionnaires urbains d'interagir avec le système en langage naturel, de
modéliser les processus métier sous forme d'automates, et de générer automatiquement des
rapports analytiques.
Votre mission est de concevoir et développer un système intégré qui combine :
• Une base de données relationnelle pour les données urbaines
• Un compilateur de requêtes en langage naturel vers SQL
• Des automates à états finis pour modéliser les workflows métier
• Un module d'IA générative pour la production de rapports
Objectifs Pédagogiques
1. Appliquer les concepts de théorie des langages (abordés dans le cours)
2. Modéliser des processus complexes avec des automates à états finis
3. Intégrer des techniques d'IA générative dans un système réel
4. Développer une architecture logicielle complète (front-end, back-end, base de
données)
5. Travailler sur des données urbaines réelles (simulées)

Partie 2 : Développement:
2.1 Moteur de Compilation (NL → SQL)
Implémenter en Python un compilateur qui transforme les requêtes en langage naturel en
requêtes SQL.
Fonctionnalités attendues :
• Tokenisation (lexer)
• Analyse syntaxique selon votre grammaire (parser)
• Construction d'un AST (Abstract Syntax Tree)
• Génération de code SQL
• Gestion des erreurs syntaxiques/sémantiques
2.2 Moteur d'Automates
Implémenter les automates définis en 1.2 avec capacité à :
• Vérifier la validité d'une séquence d'événements
• Déterminer l'état courant d'une entité (capteur, intervention, véhicule)
• Déclencher des actions automatiques (ex: alerte si capteur hors service > 24h)
Technologie : Utiliser la librairie transitions ou implémenter votre propre moteur.
